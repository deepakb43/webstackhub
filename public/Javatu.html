Introduction to Java

Java is a popular, versatile, and powerful programming language used to build applications for many platforms including web, mobile, and desktop.

This tutorial is designed for absolute beginners with no prior programming or computer science experience. We will start from the very basics and gradually build your understanding of Java programming.

Java is known for its "write once, run anywhere" capability, which means that Java programs can run on any device that has a Java Virtual Machine (JVM) installed. This makes Java a great choice for building applications that need to work across different types of devices and operating systems.

Java is easy to learn even without a computer science background. It is widely used in many industries, from Android apps to enterprise systems, offering many opportunities to create useful software.

Let's get started on your programming journey!

Variables & Data Types

Variables are containers that hold data values. In Java, every variable has a type that determines what kind of data it can store.

  • int: stores whole numbers (e.g., 5, 100)
  • double: stores decimal numbers (e.g., 3.14, 0.5)
  • char: stores a single character (e.g., 'a', 'Z')
  • boolean: stores true or false values
  • String: stores a sequence of characters (text)

Control Flow

            
Control flow statements allow you to control the order in which your code executes.

            
    
                
  • if-else: executes code based on a condition
    • 
                
  • for loop: repeats code a specific number of times
    • 
                
  • while loop: repeats code while a condition is true
    • 
            


            

                

                    
Example: if-else Statement

                    

                        
                    

                

                

                    
int number = 10;
if (number > 0) {
    System.out.println("Positive number");
} else {
    System.out.println("Zero or negative number");
}

                

            

        


        

            
Methods

            
Methods are blocks of code that perform a specific task and can be reused.

            
They help organize your code and make it more readable.


            

                

                    
Example: Defining and Calling a Method

                    

                        
                    

                

                

                    
public class Main {
    public static void main(String[] args) {
        greet();
    }

    public static void greet() {
        System.out.println("Hello, welcome to Java!");
    }
}

                

            

        


        

            
Object-Oriented Programming

            
Java is an object-oriented language, which means it uses objects to represent data and methods.

            
Object-Oriented Programming (OOP) is a way of organizing your code to make it easier to understand, reuse, and maintain. It is based on a few simple ideas:

            
    
                
  • Class: Think of a class as a blueprint or a recipe. It defines what an object will be like, including its properties (data) and behaviors (methods).
    • 
                
  • Object: An object is like a real-world thing created from the class blueprint. It has its own values for the properties and can perform actions defined by the class.
    • 
                
  • Inheritance: This allows a new class to use the features of an existing class. It helps you avoid repeating code by building on what already exists.
    • 
                
  • Encapsulation: This means keeping the details inside an object hidden from the outside world, only showing what is necessary. It helps protect the data and makes the code easier to use.
    • 
                
  • Polymorphism: This means "many forms." It allows objects to be treated as instances of their parent class, letting you use the same method in different ways.
    • 
                
  • Abstraction: This means focusing on the important things and hiding the complex details. It helps you think about what an object does, not how it does it.
    • 
            

            
Using OOP has many benefits:

            
    
                
  • Modularity: Your code is divided into small, manageable pieces (classes and objects).
    • 
                
  • Reusability: You can reuse existing code by creating new classes based on old ones.
    • 
                
  • Maintainability: It is easier to update and fix your code because it is organized and clear.
    • 
            


            

                

                    
Example: Simple Class and Object

                    

                        
                    

                

                

                    
public class Dog {
    String name;

    public Dog(String name) {
        this.name = name;
    }

    public void bark() {
        System.out.println(name + " says Woof!");
    }
}

public class Main {
    public static void main(String[] args) {
        Dog myDog = new Dog("Buddy");
        myDog.bark();
    }
}
